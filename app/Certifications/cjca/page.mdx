## exam structure: what you're actually doing

---

<Callout type="error"> ⚔️ Red Team </Callout>
- 5 machines. 
- get User + Root on each. 
- 10 pts per flag. 

straightforward if you've been solving machines regularly. don't overthink the priv esc — the module covers it.

<Callout type="info"> ⚔️ Blue Team </Callout>

this is the part people sleep on and shouldn't.

- 40 alerts in the queue. 
- you need to analyze around 27 of them. 
- for each one, you mark it **FP (False Positive)** or **TP (True Positive)** and write a clear explanation.

the cool part? **the traces from your own red team actions show up in the blue team alerts.** you're literally hunting yourself. which means if you did something on day 1, you should be able to explain exactly what alert it triggered and why on day 4.

<Callout type="info">
  that correlation between red and blue is what makes CJCA actually interesting vs just another box-popping cert.
</Callout>


## reporting: don't sleep on SysReptor

<Callout>
  read this before the exam: [HTB CJCA Sample Report (SysReptor)](https://docs.sysreptor.com/assets/reports/HTB-CJCA-Report.pdf)
</Callout>

- install SysReptor before exam day. set up a CJCA template.
- every finding = screenshot + reproducible steps. no exceptions.
- if you know markdown, SysReptor becomes comfortable fast — but you need to learn the UI first, not mid-exam.
- collect outputs and screenshots as you go. going back to reconstruct them under stress is rough.

## what about AI?
 
honestly? it comes down to your ethics.
 
i didn't care about using AI during the exam. i've always seen HTB as a learning opportunity first — the cert is just a byproduct of actually getting good. if you're using AI to skip the understanding, you're only cheating the learning, not the exam.
 
<Callout type="info">
  if you're doing this to actually get better at security — use AI as a tool, not a crutch. if you're doing this just to collect a badge, that's a you problem.
</Callout>
