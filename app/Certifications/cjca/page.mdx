## my prep (honest version)

i did the CJCA learning path on HTB. that's the baseline requirement and i'm not gonna pretend i did anything extra fancy.

background context: i'm not really a blue team person. haven't explored it much. BUT i did a diploma project around the ELK stack so i wasn't going in completely blind. As for the red team part, i was already solving machines regularly so nothing changed there.

<Callout type="warning">

</Callout>

## exam structure: what you're actually doing


<Callout type="error"> ⚔️ Red Team </Callout>
- 5 machines. 
- get User + Root on each. 
- 10 pts per flag. 

straightforward if you've been solving machines regularly. don't overthink the pentest in nutshell module covers it.

<Callout type="info"> ⚔️ Blue Team </Callout>

this is the part people sleep on and shouldn't.

- 40 (approximately) alerts in the queue. 
- you need to analyze them.
- for each one, you mark it **FP (False Positive)** or **TP (True Positive)** and write a clear explanation with backing evidence from ELK machine.

## reporting: don't sleep on SysReptor

<Callout>
  read this before the exam: [HTB CJCA Sample Report (SysReptor)](https://docs.sysreptor.com/assets/reports/HTB-CJCA-Report.pdf)
</Callout>

- install SysReptor before exam day. set up a CJCA template.
- every finding = screenshot + reproducible steps. no exceptions.
- if you know markdown, SysReptor becomes comfortable fast, but you need to learn the UI first, not mid-exam unlike someone who's writing this blog xD.
- collect outputs and screenshots as you go. going back to reconstruct them under stress is rough 😕.

## what about AI?
 
honestly? it comes down to your ethics.
 
i didn't care about using AI during the exam. i've always seen HTB as a learning opportunity first the cert is just a byproduct of actually getting good. if you're using AI to skip the understanding, you're only cheating the learning, not the exam. Imagine paying for a exam and using AI to cheat the exam? 😭
 
<Callout type="info">
  if you're doing this to actually get better at security use AI as a tool, not a crutch. if you're doing this just to collect a badge, that's a you problem.
</Callout>


## tldr for people who scrolled here first

- do the HTB path but don't stop there solve machines in HTB labs starting point section, do extra ELK labs
- blue team is where people get caught off guard. take it seriously
- the report is not optional fluff. it's the whole thing
- sleep, eat, gym during the exam. recovery is a strategy
- learn SysReptor before day one. set up the template. read the sample report
- trust your prep. the self-doubt spiral is real and a waste of energy
- **Believe** in yourself. You can do it.
